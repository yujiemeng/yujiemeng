Shneiderman的八项黄金规则

2017-06-19

Shneiderman的八项黄金规则

作者：EUPHEMIA WONG

如果您想要设计出优秀，高效和无挫的用户界面，请遵循本·谢内德曼（Ben Shneiderman）的“八个黄金界面设计规则”。 苹果，谷歌和微软其精心设计的产品反映了Shneiderman的规则。可以在诸如上述企业巨头制作的各种用户界面中可以发现Shneiderman的黄金规则。这些规则的视觉实现在其产生的流行界面中更加明显。本文将教你如何通过整合8项黄金规则来优化您的工作。

8 Golden Rules of Interface Design

Ben Shneiderman（生于1947年8月21日）是美国马里兰大学人机交互实验室的计算机科学家和教授。他的作品与唐诺曼（Don Norman）和雅各布·尼尔森（Jakob Nielsen）等当代设计思想家相当。他的热门书《设计用户界面：有效的人机互动策略》揭示了他八个接口设计的黄金规则：

  • 在相似的用户场景和操作顺序时要利用用户熟悉的图标，颜色，菜单层次结构，呼叫动作和用户流来努力实现一致性。规范信息传递的方式，确保用户能将熟悉的信息从一次点击移植到另一次点击中。而不需要为相同的动作学习新的规则。一致性通过帮助用户熟悉产品的数字化环境起着重要的作用，从而更轻松地实现目标。

  • 让用户启用快捷方式。随着使用的增加，需要提供更快完成任务的方法。例如，Windows和Mac为用户提供了用于复制和粘贴的键盘快捷键，以便用户变得更有经验时，他们可以更快速，轻松地浏览和操作用户界面。

  • 提供信息反馈。用户应该知道他们在哪里，什么时候发生什么。 对于每一个行动，在合理的时间内都应该有适当的，人性化的反馈。 应用这个的一个很好的例子是，在通过多页问卷调查的过程中向用户指出他们在哪里。我们经常看到的一个不好的例子是，当有错误消息显示时，是错误的代码而不是人类可读和有意义的消息。

Windows Media Player设计师应该记住Ben Shneiderman的第三个黄金规则：提供信息反馈。错误消息显示一个错误代码，对用户来说并不意味着什么。但作为一名好的设计师，您应该始终致力于给用户以可读和有意义的反馈。

  • 设计要闭环。不要让你的用户猜测。用户的行动将会得到什么样的反馈。例如，用户在完成在线购买后会收到“谢谢”消息和购买收据凭证。
  • 提供人性化的错误提示。没有人喜欢被告知他们错了，特别是你的用户。系统应设计的尽可能简单，但是当出现不可避免的错误时，确保向用户提供简单，直观的分步说明，以尽快和无痛地解决问题。例如，标记用户忘记填写在线的表单字段。
  • 允许容易地撤销动作。设计人员应旨在为用户提供明显的方式来撤销其行为。用户应该在不同点允许这些反转，无论是单次动作，数据输入还是整个动作场景。 正如斯内德曼在他的书中所说：

这个功能减轻了用户操作时的焦虑，因为用户知道错误可以被撤销; 因此，这将鼓励用户探索不熟悉的选项。

  • 支持内部控制点。让您的用户成为行动的发起者。给用户感觉他们完全控制应用中发生的事件。在设计系统时按照预期的方式获得用户的信任。

  • 减少短期记忆的负荷。人的注意力是有限的，我们只能在一次短期记忆中保留五个项目。因此，界面应尽可能简单，具有适当的信息层次，并能够使用户对场景返回有识别。认识到一些事情总是比回想起来容易，因为认知过程涉及到感知线索，帮助我们进入我们的广泛的记忆，并允许相关信息的表现。例如，我们经常发现多项选择问题的格式比简短的问答题容易，因为它只需要我们认识到答案，而不是从我们的记忆中回忆起来。被彭博商业周刊称为“世界上最具影响力的设计师”之一的用户倡导者雅各布·尼尔森（Jakob Nielsen）发明了几种可用性方法，包括启发式评估。识别召回同样是是Nielsen的10个可用性启发式接口设计之一。

学习苹果是如何整合Shneiderman的八项黄金规则的

苹果公司是在北美的大型技术公司，是反映Shneiderman八项黄金规则的设计如何导致成功产品的一个很好的例子。该公司从Macintosh到移动设备的各个方面取得了巨大的成功。他们以一直以直观而美丽的设计为荣。苹果的iOS人机接口指南在2014年年中公布，从中可以看出他们的设计团队如何应用像Shneiderman这样的设计原则。

1.一致性

“一致性”和“感知稳定性”被编入苹果Mac系统的设计准则中。Mac系统的菜单栏包含一致的图形元素，无论是从1980年代还是2010年的版本。

两个年代的Mac系统的外观，它的系统菜单栏一直保持一致性。

2. 快捷键

如前所述，Mac允许用户使用各种键盘快捷键，常用来举例的包括复制和粘贴（Command-X和Command-V）以及截图（Command-Shift-3）。

Mac允许用户通过为他们提供键盘快捷方式放弃鼠标点击。

3. 信息反馈

当用户点击Mac桌面上的文件时，文件的“突出显示”可以让用户看到一个很好的视觉反馈。另一个例子是当用户拖动桌面上的文件夹时，他们可以看到在按住鼠标时物理上被移动的文件夹。

当用户点击“Learning”文件夹时，“Learning”文件夹变为突出显示。

当用户按住鼠标并拖动桌面上的文件夹时，文件夹表示为物理移动。

4. 对话

当用户将软件安装到Mac OS时，信息量大的屏幕显示用户当前在安装的步骤。


版权所有：Google；版权协议：正常使用

当用户安装程序“Parallels Desktop 9”时，系统展示它正处于拷贝文件过程。

5. 错误处理

在软件安装过程中，如果发生错误，通过提示信息用户将友好地收到警报。这里重要的是要知道来根据当前错误的严重程度告知用户，何时使用较小的，较少侵入性的警报，何时使用更大的警报。然而，当错误发生时对用户进行惩罚几乎是不可接受的，所以用户阅读的错误信息时，请谨慎选择正确的语气和正确的语言。所以不要简单地留下一个错误代码来“处理”它！

显示一个温和的错误消息，向用户解释发生了什么以及为什么发生。通过解释这是由于自己的安全偏好选择导致，并且进一步向用户保证这些是可控的（参见下面的“支持内部控制位置”）。

Windows的一个坏例子：显示一个使用“致命”和“终止”字样的错误消息。 这样的负面，不友好的话肯定会吓倒大多数用户！

6. 允许撤销动作

当用户在安装过程中提供发生错误，他们被允许回到上一步，而不是因为错误而必须“受到惩罚”从头重新做操。

用户可以非常简单和快速的撤销上一步操作。

7.支持内部控制点

让用户有权力选择是继续运行程序还是退出程序。Mac的活动监视器允许用户在程序意外崩溃时“强制退出”。

用户可以退出一个程序，或者当一个程序崩溃时强制推出。

8. 减少短时记忆负荷

由于人类只能在短期记忆中内保留5个项目，苹果iPhone在屏幕底部的主菜单区域只能放置4个应用程序图标。该决定不仅涉及对记忆负载的考虑，还兼顾了一致性。

通过在不同版本的iOS上显示相同的底层菜单，这是Apple如何实现一致性规则（第一条规则）的很好的例子。这也是苹果如何减少用户短期记忆负载（第八条规则）的一个很好的例子。由于人类只能在一段时间内保留5项短期记忆，所以苹果iPhone已经在屏幕底部的主菜单区域中只能放置4个应用程序图标，无论是iOS 4 或iOS 7。

总结

当您遵循Ben Shneiderman的“八金接口设计规则”时，您将会像Apple，Google和Microsoft一样设计出卓越，富有成效和无懈可击的用户界面。从Mac和PC到移动设备或虚拟现实以及将来发明的任何其他互动技术，只要您的设计涉及到人与界面之间的交互，这八项黄金规则在设计过程中至关重要，不容错过。要开始使用，请使用附加的对照表学习将这些规则应用于您的工作。
